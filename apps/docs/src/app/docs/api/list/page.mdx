# List Endpoints

The List Endpoints provide access to comprehensive lists of various financial instruments including stocks, ETFs, cryptocurrencies, forex pairs, and available indexes. These endpoints are useful for discovering available symbols and building comprehensive financial applications.

## Working with List Data

### Filtering and Searching

List endpoints return comprehensive data that you can filter and search through:

<CodeBlock language="typescript">
{`// Get all lists
const [stocks, etfs, cryptos, forex, indexes] = await Promise.all([
    fmp.list.getStockList(),
    fmp.list.getETFList(),
    fmp.list.getCryptoList(),
    fmp.list.getForexList(),
    fmp.list.getAvailableIndexes()
]);

// Filter stocks by exchange
const nasdaqStocks = stocks.data.filter(stock =>
stock.exchangeShortName === 'NASDAQ'
);

// Search for specific symbols
const searchSymbol = (lists: any[], symbol: string) => {
return lists.flatMap(list =>
list.data.filter((item: any) =>
item.symbol.toLowerCase().includes(symbol.toLowerCase())
)
);
};

const appleResults = searchSymbol([stocks, etfs], 'AAPL');
console.log('Apple-related symbols:', appleResults);`}

</CodeBlock>

### Building Symbol Lookups

Create efficient symbol lookups for your applications:

<CodeBlock language="typescript">
{`// Create a comprehensive symbol lookup
const createSymbolLookup = async () => {
    const [stocks, etfs, cryptos, forex, indexes] = await Promise.all([
        fmp.list.getStockList(),
        fmp.list.getETFList(),
        fmp.list.getCryptoList(),
        fmp.list.getForexList(),
        fmp.list.getAvailableIndexes()
    ]);

    const symbolLookup = new Map();

    // Add stocks
    stocks.data.forEach(stock => {
        symbolLookup.set(stock.symbol, {
        ...stock,
        type: 'stock'
        });
    });

    // Add ETFs
    etfs.data.forEach(etf => {
        symbolLookup.set(etf.symbol, {
        ...etf,
        type: 'etf'
        });
    });

    // Add cryptocurrencies
    cryptos.data.forEach(crypto => {
        symbolLookup.set(crypto.symbol, {
        ...crypto,
        type: 'crypto'
        });
    });

    // Add forex pairs
    forex.data.forEach(pair => {
        symbolLookup.set(pair.symbol, {
        ...pair,
        type: 'forex'
        });
    });

    // Add indexes
    indexes.data.forEach(index => {
        symbolLookup.set(index.symbol, {
        ...index,
        type: 'index'
        });
    });

    return symbolLookup;

};

// Usage
const lookup = await createSymbolLookup();
const aaplInfo = lookup.get('AAPL');
console.log('AAPL info:', aaplInfo); // { symbol: 'AAPL', name: 'Apple Inc.', type: 'stock', ... }`}

</CodeBlock>

### Exchange-Based Filtering

Filter instruments by specific exchanges:

<CodeBlock language="typescript">
{`// Get all instruments by exchange
const getInstrumentsByExchange = async (exchange: string) => {
    const [stocks, etfs, cryptos, forex, indexes] = await Promise.all([
        fmp.list.getStockList(),
        fmp.list.getETFList(),
        fmp.list.getCryptoList(),
        fmp.list.getForexList(),
        fmp.list.getAvailableIndexes()
    ]);

    const allInstruments = [
        ...stocks.data.map(item => ({ ...item, type: 'stock' })),
        ...etfs.data.map(item => ({ ...item, type: 'etf' })),
        ...cryptos.data.map(item => ({ ...item, type: 'crypto' })),
        ...forex.data.map(item => ({ ...item, type: 'forex' })),
        ...indexes.data.map(item => ({ ...item, type: 'index' }))
    ];

    return allInstruments.filter(instrument =>
        instrument.exchangeShortName === exchange
    );

};

// Get all NASDAQ instruments
const nasdaqInstruments = await getInstrumentsByExchange('NASDAQ');
console.log('NASDAQ instruments:', nasdaqInstruments.length);`}

</CodeBlock>

## Error Handling

Always check the success property before accessing data:

<CodeBlock language="typescript">
{`const stockList = await fmp.list.getStockList();

if (stockList.success) {
console.log('Total stocks:', stockList.data.length);

    // Process the data
    stockList.data.forEach(stock => {
        console.log(\`\${stock.symbol}: \${stock.name}\`);
    });

} else {
console.error('Error fetching stock list:', stockList.error);
console.error('Status:', stockList.status);
}`}

</CodeBlock>

## Performance Considerations

### Caching List Data

Since list data doesn't change frequently, consider caching the results:

<CodeBlock language="typescript">
{`class CachedListManager {
    private cache = new Map();
    private cacheExpiry = new Map();
    private readonly CACHE_DURATION = 24 * 60 * 60 * 1000; // 24 hours

    private isCacheValid(key: string): boolean {
        const expiry = this.cacheExpiry.get(key);
        return expiry && Date.now() < expiry;
    }

    async getStockList() {
        const cacheKey = 'stockList';

        if (this.isCacheValid(cacheKey)) {
        return this.cache.get(cacheKey);
        }

        const result = await fmp.list.getStockList();

        if (result.success) {
        this.cache.set(cacheKey, result);
        this.cacheExpiry.set(cacheKey, Date.now() + this.CACHE_DURATION);
        }

        return result;
    }

    // Similar methods for other list types...

}

// Usage
const listManager = new CachedListManager();
const stocks = await listManager.getStockList(); // Cached after first call`}

</CodeBlock>

## Rate Limiting

List endpoints are subject to FMP's rate limits. For production applications:

- Cache list data when possible
- Implement appropriate rate limiting
- Consider pagination for large datasets
- Use bulk operations when available
